lit is a local-only, offline-only version control prototype. It borrows the basic shape of Git for one-computer workflows, without remotes, accounts, servers, or cloud features.

What is lit?

lit means "local git". The goal is simple: keep local checkpoints on one machine with a workflow that feels familiar to Git users while staying much smaller in scope.

  • local-only
  • offline-only
  • single-computer workflow
  • deterministic on-disk storage

Install and setup

lit currently targets Python 3.12 or newer.

git clone https://github.com/Ahnd6474/lit.git
cd lit
python -m venv .venv
pip install -e .
python -m lit init

If you prefer a local static server for these docs, run python -m http.server inside website/ and open the local address it prints.

Planned command workflow

This is the beginner-friendly path lit is being built toward. In this revision, it is a roadmap, not a fully working demo. 

lit init
lit add notes.txt
lit commit -m "save first draft"
lit branch experiment
lit checkout experiment
lit merge experiment
lit rebase main
lit restore notes.txt

Today, only the first line is executable. The rest describe the intended Git-like workflow for future milestones.

Git: same ideas, smaller scope

Similar

  • repository per folder
  • hidden metadata directory
  • planned staging, commit, branch, merge, and rebase flow
  • content-addressed objects

Different

  • no push, pull, fetch, clone, or remotes
  • no accounts or hosted service
  • no collaboration model
  • designed to stay local-only and offline-only

Limitations and non-goals

  • The repository is still in the bootstrap stage.
  • Staging, commit history, diff, restore, branch switching, merge, and rebase are not implemented yet.
  • Remote workflows are not planned.
  • Cloud sync, accounts, and network-dependent features are out of scope.
