lit

lit means "local git." It is a lightweight, local-only, offline-only version control prototype for one computer.

Quick start Commands Limitations

What lit is

lit is a CLI-first project for local checkpoints and Git-like version control workflows without any server, account, remote, sync service, or network dependency.

The design goal is narrow on purpose: keep local history simple, deterministic, and easy to inspect inside a .lit/ directory.

Why local-only

lit is built for one machine. It does not support collaboration, remotes, or hosted workflows.

Why offline-only

lit should stay usable without internet access. No login, no cloud, and no network call should be required to use it.

Installation and local setup

lit currently targets Python 3.12 or newer.

python -m venv .venv
.venv\Scripts\Activate.ps1
pip install -e .
lit --help

You can also run the module form directly: python -m lit --help

Quick start

python -m lit init .
python -m lit init my-project

Successful bootstrap creates local metadata in .lit/, including config.json, HEAD, index.json, object folders, refs, and merge or rebase state files.

Beginner-friendly example workflow

  1. Run python -m lit init . in your project folder.
  2. Edit or add files.
  3. Use lit add when staging support lands.
  4. Use lit commit to create checkpoints once implemented.
  5. Create a side branch with lit branch.
  6. Bring work together with lit merge or lit rebase.
  7. Use lit restore or lit checkout to recover or move around history.

Today only the first step is available. The other commands are part of the intended workflow, but they are not implemented yet.

Similar to Git

  • CLI-first workflow
  • Hidden repository metadata
  • Git-like command names
  • Planned branch, merge, and rebase flow

Different from Git

  • local-only by design
  • offline-only by design
  • no remotes, push, pull, fetch, or clone
  • smaller scope and earlier implementation stage

Current limitations and non-goals

  • Only lit init is implemented today.
  • The staged-file and commit workflow is not implemented yet.
  • Branch switching, merge execution, and rebase execution are not implemented yet.
  • There are no remotes, no hosted services, and no collaboration features.
  • The site is static HTML and CSS so it can be opened directly or served locally without heavy tooling.
