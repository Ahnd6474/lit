local execution vcs

lit is a local execution VCS for autonomous coding workflows on one machine.

Use it for safe checkpoints, rollback, provenance, verification-aware history, lineage isolation, and local artifacts. No account, no server, no sync, no remote, and no network requirement.

Quick Start Limitations

Why use lit?

`lit` keeps a few familiar Git-like verbs for local work, but the product centers safe checkpoints, rollback, provenance, verification, and lineage isolation for long-running autonomous workflows on one machine.

  • Local-only and offline-only by design
  • Safe checkpoints and rollback anchors
  • Structured provenance and verification status
  • Lineage isolation for parallel local workers
  • CLI and optional desktop GUI on one shared backend
  • No `push`, `pull`, `fetch`, `clone`, or hosted service
  • No account, login, cloud, daemon, or background sync

Installation

`lit` targets Python 3.12+.

python -m pip install -e .

That installs the `lit` command from this repository. If the script directory is not on your PATH, use python -m lit ....

Quick start

mkdir demo
cd demo
lit init
python -c "from pathlib import Path; Path('note.txt').write_text('hello\n', encoding='utf-8')"
lit add note.txt
lit commit -m "Create first checkpoint"
lit checkpoint create --name safe-start
lit status
lit log
lit doctor

Command overview

Command What it does
lit init [path]Create a new .lit repository.
lit add <paths...>Stage files or directories.
lit commit -m "message"Create a commit from the index.
lit statusShow staged, modified, deleted, and untracked files.
lit diffShow the working tree diff against the current commit.
lit logShow first-parent history from HEAD.
lit restore [paths...] --source <rev>Restore tracked files without moving HEAD.
lit checkout <branch-or-commit>Switch branches or detach HEAD.
lit branch [name]List branches or create a new branch.
lit merge <rev>Merge another local revision into the current branch.
lit merge --abortClear merge state and restore the pre-merge tree.
lit rebase <rev>Rebase the current branch onto another local revision.
lit rebase --abortClear rebase state and restore the original branch tip.
lit checkpoint ...Create, list, inspect, and pin safe checkpoints.
lit rollback ...Restore the latest safe checkpoint or a selected checkpoint.
lit verify ...Run or replay verification and inspect cached status.
lit lineage ...Create, inspect, switch, preview, promote, or discard lineages.
lit artifact ...Inspect artifacts, linkage, and storage usage.
lit doctorInspect health, locks, and unfinished transactions.
lit exportBuild a Git-facing export plan with provenance mapping.

Example workflow

lit init
lit add src
lit commit -m "Save working version"
lit checkpoint create --name safe-base
lit lineage create worker-a --owned-path src/lit
lit lineage switch worker-a
lit verify status
lit lineage switch main
lit lineage promote worker-a --preview
lit rollback

Git similarities and differences

Similar to Git:

  • Staging before commit
  • Local history and branches
  • Merge, rebase, checkout, and detached HEAD

Different from Git:

  • No remote repositories
  • No `push`, `pull`, `fetch`, or `clone`
  • No collaboration or team workflow
  • Safe checkpoints and rollback are first-class
  • Verification, provenance, lineages, and artifacts are product features

Local and offline constraints

`lit` is meant for one machine. Every workflow is local. Nothing in the current product design depends on a network connection.

  • No server process
  • No cloud storage
  • No account system
  • No synchronization between computers

Limitations and non-goals

  • `lit export` is a Git bridge, not a full Git transport or object-compatibility layer.
  • Conflict handling is manual: `lit` writes markers and preserves state.
  • The desktop UI is optional and requires the gui dependency extra.
  • The product is intentionally scoped to one-machine local execution workflows.
  • Remote hosting, collaboration, cloud sync, and network workflows are non-goals.
